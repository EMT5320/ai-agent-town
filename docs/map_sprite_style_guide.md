# 地图小人风格指南：二次元 Q 版低像素

> 本指南用于 `Agent Valley` 地图层小人的返工准备。当前只做文档与提示词准备，不生成新图片，不修改 Godot 代码，也不把任何既有 `pending_review` 小人提升为 `source_selected`。

## 目标定位

地图小人的统一定调为：**二次元 Q 版低像素地图小人**。

目标效果：角色在 Godot 地图层移动时，能用很小尺寸读出“谁是谁”，同时继承当前已通过的二次元轻幻想田园主视觉。地图小人服务可玩 Demo 的地图识别，表情魅力继续由 VN 半身立绘承担。

## 尺寸与交付

| 阶段 | 规格 |
| --- | --- |
| 生图画布 | 方图，推荐 512x512 或 1024x1024；单角色居中；透明背景优先 |
| 抠图兜底 | 若生成器不支持透明背景，使用纯 #ff00ff 或纯白背景，角色内部不能使用该背景色 |
| 后处理目标 | `64x64` PNG，透明背景 |
| 角色安全框 | 主体约 `48x56`，左右各留约 8 px，底部留约 4 px 落脚余量 |
| 首版状态 | 只做 `idle_front` 静态图 |
| 未来扩展 | `64x64` 单元格 Sprite Sheet；方向可扩展为 down / left / right / up，每方向 3 帧 |

## 头身比例

- 约 `2.0-2.5` 头身。
- 头部占全高 44%-50%，发型轮廓必须优先清楚。
- 身体短、四肢简化，手脚可以是小块面。
- 道具只保留一个核心识别点，尺寸不能压过头部和躯干。

## 视角

- 使用轻俯视正面，镜头高约 20-30 度。
- 脚底和身体重心要清楚，方便放在俯视 / 斜俯视地图上。
- 首版避免正侧面、背面、战斗待机姿势、舞台表演姿势和 3D 模型角度。

## 轮廓线

- 外轮廓使用 1-2 px 深色线，推荐深棕、深蓝或深紫，避免纯黑厚边。
- 内部线条只保留发型分区、上衣边界、道具轮廓等关键识别点。
- 缩小到 `64x64` 后轮廓不能糊成一团；继续缩到 `32x32` 仍要能看出主色和身份。

## 调色

- 每个角色保留 3-5 个主色：发色、服装主色、服装辅色、道具色、少量星灯点缀。
- 阴影 1-2 级即可，保持块面清楚。
- 低魔法元素只做小面积星灯、发饰、吊坠或道具纹样，不能变成大面积发光特效。
- 避免复杂渐变、脏噪点、重 dithering、金属反光和过度材质细节。

## 生成失败判定

出现以下任一情况，本轮图不能进入 `source_selected`：

- 画面成了半身头像、VN 立绘、写实全身插画、3D 模型或侧卷轴角色。
- 角色超过 3 头身，头部识别弱，Q 版地图小人感不足。
- 视角与地图不匹配：纯正面纸片人、正侧面、强透视、舞台 pose、战斗 pose。
- 轮廓线太糊、抗锯齿太重、细节太碎、缩到 `64x64` 看不清。
- 背景复杂，透明或抠图不可用。
- 身份继承失败：主色、发型、职业道具或年龄气质偏离继承来源。
- 玩家和米娅看起来像同一人；奥蕾娅明显年轻化；托玛不像首发唯一男性木匠；凯娅变现代偶像；布兰娜失去成熟劳动者体感。
- 出现文字、水印、logo、额外人物、残肢、错手错脚、严重透视错误。

## 第一批试生成范围

先做最小可比对批次：

1. `player_farmer_map_idle`：验证玩家主色、农场主身份和 Q 版比例。
2. `npc_mira_map_idle`：验证女性 NPC、暖橙杂货铺身份与玩家区分度。
3. `npc_tomas_map_idle`：验证男性 NPC、深蓝木匠身份和工具道具读感。

三张一起看 `64x64` 与 `32x32` 缩略图。若比例、轮廓和身份继承都稳定，再生成 `npc_orren_map_idle`、`npc_lena_map_idle`、`npc_kai_map_idle`、`npc_bram_map_idle`。交互标记可以等人物小人的线宽和调色稳定后再做。
