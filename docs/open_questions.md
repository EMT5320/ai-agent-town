# 待确认问题

本文记录需要主人拍板或继续讨论的问题。未确认的问题不阻塞阶段 0 技术 Spike。

## 1. 主客户端是否正式选 Godot

当前建议：

- 阶段 0 使用 Godot 做 Spike。
- 若 2～3 天内能跑通玩家移动、NPC 显示、后端通信和对话 UI，就正式采用 Godot。

需要确认：

- 主人是否接受学习 Godot / GDScript 的成本。
- 首发目标是否以 Windows 桌面 Demo 为主。
- Web 展示是否放到后续阶段。

## 2. 游戏视觉风格

候选方向：

1. 温暖绘本风生活模拟。
2. 精致 2.5D 微缩小镇。
3. 像素风加高清角色立绘。
4. 手绘水彩风。

需要确认：

- 作品集更偏“精美插画感”还是“完整游戏感”。
- 角色是否需要像素小人动画。
- 事件 CG 是否作为首版强展示点。

## 3. 首版 NPC 数量与复杂度

当前建议：

- 6 个 NPC。
- 每个 NPC 都有完整人设、日程、关系和记忆。

需要确认：

- 是否坚持 6 个高质量 NPC。
- 是否保留原项目 10 个 NPC 设定作为后续扩展。

## 4. 玩家身份

候选方向：

1. 新搬来的农场主。
2. 小镇观察员。
3. 失忆旅人。
4. 研究员伪装成普通居民。

当前建议：

- 首版使用“新搬来的农场主”，最贴近生活模拟游戏体验。
- Debug 控制台保留“研究员视角”。

需要确认：

- 玩家在剧情中是否知道 NPC 是 Agent。
- 玩家是否拥有开发者干预能力。

## 5. LLM Provider 策略

当前建议：

- 后端保留 RuleBasedProvider。
- 云端 Provider 用于关键对话、重大事件和夜间反思。
- 模型配置继续走 `config/models.json` 与本地密钥覆盖。

需要确认：

- 第一阶段是否默认规则 Provider 跑通。
- 云端 Provider 是否在 Spike 后立即接入。
- 每次 Demo 可接受的调用成本范围。

## 6. 论文重点

候选重点：

1. LLM NPC 在生活模拟游戏中的自主行为生成。
2. 多 Agent 游戏世界中的主观记忆管理。
3. 工具调用式 NPC 行为系统的可解释性。
4. AI 辅助美术资产生产在独立游戏原型中的应用。

当前建议：

- 主线放在“主观记忆管理 + 工具调用式行为可解释性”。
- 美术资产管线作为辅助章节或工程亮点。

需要确认：

- 论文更偏游戏 AI、Agent 系统，还是人机交互体验。

## 7. 当前项目是否原地演进

当前建议：

- 在当前仓库原地演进。
- 保留现有 Python Agent Runtime。
- 新增 `clients/godot/`。
- 旧 `frontend/` 后续迁移到 `web-admin/`。

需要确认：

- 是否继续沿用 `ai-agent-town-lab` 仓库名。
- 是否把新项目对外命名为 `Agent Valley`。
